AutoValuationPro API Documentation

Comprehensive details for interacting with the AutoValuationPro car valuation API

Step 1: API Overview

The AutoValuationPro car valuation API allows you to send and retrieve car valuation data for your clients. You can create requests for car valuations by submitting client details and receiving valuation responses.

Step 2: API Endpoint

URL: https://autovaluationpro.arybit.co.ke/api/

Method: POST

Authentication:API Key(Ensure you include a valid API key in the request body).

You can get an API Key at:

  • https://autovaluationpro.arybit.co.ke/

This endpoint handles both the request creation and response retrieval. You can submit client details, including logbook images or vehicle registration numbers, and the API will process the request, returning details about the valuation.

Step 3: Request Body

The request body is in JSON format and includes details about the client, the vehicle registration number, or logbook image for the vehicle being valued. Below are two examples of the request body format:

Example 1: Submit Client Information and Logbook Image
{
    "APIKey": "api-key",
    "client-name": "Client Name",
    "client-contact": "Client Contact",
    "client-email": "client-email@example.com",
    "client-kra": "Client KRA",
    "logbookPhotoToUpload": [
        "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAOsAAADECAYAAAB......."
    ]
}

Make sure to replace the placeholder values with real data when submitting requests.

Example 2: Submit Registration Number
{
    "APIKey": "api-key",
    "registration-no": "KDA123Z"
}

This example uses the vehicle’s registration number to request valuation data.

Step 5: Common Errors

Here are some common errors you might encounter while using the API:

  • Invalid API Key: Ensure the API key provided is correct.
  • Missing Required Fields: Check if all required fields like client name, email, and logbook photo are present in the request body.
  • Unsupported File Format: Only base64-encoded images are accepted for logbook photos. Ensure the image is correctly encoded.
